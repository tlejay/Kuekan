## 1. เว็บลูกค้า (Customer Mobile Site)

Web Application ในรูปแบบ Mobile Site สำหรับผู้ใช้ทั่วไปในการสะสมแต้มจากภารกิจต่างๆ (รองรับการใช้งานผ่าน LINE Application ได้อย่างราบรื่น)

### 1.1 กระเป๋าแต้มรางวัล (KueKan Pocket)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 1.1.1 | ข้อมูลแต้ม (Points) | กระเป๋าแต้มกลางสำหรับจัดเก็บแต้มทั้งหมดที่ผู้ใช้ได้รับ | POC/MVP |
| 1.1.2 | การโอนแต้มเข้า (Inbound Transfer) | โอนแต้มจากโปรแกรมของพันธมิตรเข้ามายัง KueKan Pocket | MVP |
| 1.1.3 | การโอนแต้มออก (Outbound Transfer) | โอนแต้มจาก KueKan Pocket ไปยังโปรแกรมพันธมิตร | MVP |
| 1.1.4 | การเชื่อมต่อบัตรเครดิต (Credit Card Integration) | เชื่อมต่อกับระบบ Reward ของบัตรเครดิต (บางรายอาจรองรับเพียงการอัปโหลดหลักฐาน) | MVP/Full Scale |
| 1.1.5 | ประวัติการทำธุรกรรม (Transaction History) | แสดงประวัติการเคลื่อนไหวของแต้มทั้งหมด | MVP |
| 1.1.6 | ภาพรวมยอดคงเหลือ (Balance Overview) | แสดงยอดแต้มและมูลค่าแบบเรียลไทม์ พร้อม Data Visualization | POC/MVP |
| 1.1.7 | ระบบควบคุมปริมาณ (Volume Control) | กำหนดขอบเขตและขีดจำกัดการโอนรายวัน/รายเดือน และมีกฎตรวจสอบป้องกันการใช้งานผิดประเภท | MVP |
| 1.1.8 | ระบบการส่งออกข้อมูล (Data Export) | การส่งออกประวัติการทำธุรกรรมเป็นไฟล์ CSV หรือ PDF | MVP |

### 1.2 ศูนย์ภารกิจ (Mission Center)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 1.2.1 | ภารกิจตามตำแหน่ง (Location-Based) | เช็คอินที่โลเคชันพันธมิตรโดยใช้ระบบ Geofencing ที่แม่นยำ | POC/MVP |
| 1.2.2 | ภารกิจตามการกระทำ (Action-Based) | ภารกิจที่ต้องการการยืนยัน ตรวจสอบโดยพนักงานพันธมิตร | MVP |
| 1.2.3 | ภารกิจตามเวลา (Time-Based) | ภารกิจที่ต้องเข้าร่วมกิจกรรมภายในช่วงเวลาที่กำหนด | MVP |
| 1.2.4 | ภารกิจตามการซื้อ (Purchase-Based) | ภารกิจที่ต้องใข้จ่ายกับร้านค้าพันธมิตรพร้อมยืนยันใบเสร็จผ่าน Partner App | MVP |
| 1.2.5 | การติดตามความคืบหน้า (Progress Tracking) | แสดงสถานะและความคืบหน้าของภารกิจที่กำลังทำ | MVP |
| 1.2.6 | ระบบการแจ้งเตือน (Notifications) | แจ้งเตือนภารกิจใหม่ ภารกิจใกล้หมดอายุ หรือการอนุมัติภารกิจ | MVP |

### 1.3 ระบบแลกรางวัล (Reward Redemption)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 1.3.1 | แคตตาล็อกของรางวัล (Reward Catalog) | เลือกดูรายการของรางวัล พร้อมรายละเอียดและแต้มที่ต้องใช้แลก | MVP |
| 1.3.2 | มุมมองแผนที่ (Map View) | ดูของรางวัลบนแผนที่ตาม location ใกล้ตัว | MVP |
| 1.3.3 | การกรองตามหมวดหมู่ (Category Filtering) | ค้นหารางวัลตามประเภท (เช่น ฟรี, มาใหม่, ยอดนิยม) | MVP |
| 1.3.4 | การสร้างคูปองดิจิทัล (Digital Coupon) | สร้างคูปองพร้อม QR Code เพื่อนำไปใช้สิทธิ์ที่ร้านค้า | MVP |
| 1.3.5 | ประวัติการแลกรางวัล (Redemption History) | ดูรายการที่เคยแลกไปแล้วทั้งหมด พร้อมสถานะ | MVP |
| 1.3.6 | การติดตามสถานะคูปอง (Coupon Tracking) | ดูสถานะของคูปอง (ยังไม่ใช้/ใช้แล้ว/หมดอายุ) | MVP |
| 1.3.7 | การแจ้งเตือนใกล้หมดอายุ (Expiration Alert) | แจ้งเตือนก่อนที่คูปองหรือแต้มจะหมดอายุ | MVP |

### 1.4 ระบบระดับสมาชิก (Tiering System)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 1.4.1 | ระดับมาตรฐาน (Standard Tier) | สมาชิกเริ่มต้น พร้อมสิทธิประโยชน์พื้นฐาน | MVP |
| 1.4.2 | ระดับพิเศษ (Exclusive Tier) | สมาชิก VIP พร้อมสิทธิประโยชน์ที่มากกว่า | MVP |
| 1.4.3 | สิทธิประโยชน์ตามระดับ (Tier Benefits) | รางวัลหรือภารกิจพิเศษเฉพาะสมาชิกแต่ละระดับ | MVP |
| 1.4.4 | ความคืบหน้าของระดับ (Tier Progress) | แสดงว่าต้องสะสมอีกเท่าไหร่ถึงจะได้เลื่อนระดับ | MVP |
| 1.4.5 | เงื่อนไขการเลื่อนระดับ (Upgrade Rules) | รายละเอียดเงื่อนไขการปรับระดับสมาชิก | MVP |

### 1.5 ข้อมูลผู้ใช้และการมีส่วนร่วม (User Profile & Engagement)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 1.5.1 | โปรไฟล์ผู้ใช้ (User Profile) | แสดงข้อมูลส่วนตัว แต้มสะสม และระดับสมาชิก | POC/MVP |
| 1.5.2 | ประวัติกิจกรรม (Activity History) | ดูรายการย้อนหลังทั้งหมด (ได้รับแต้ม, ใช้แต้ม, โอนแต้ม) | MVP |
| 1.5.3 | สถิติภารกิจ (Mission Stats) | แสดงจำนวนภารกิจที่ทำสำเร็จ | MVP |
| 1.5.4 | กระดานผู้นำ (Leaderboard) | จัดอันดับผู้ใช้งาน เพื่อเพิ่มความสนุกในการแข่งขัน | MVP/Full Scale |
| 1.5.5 | การตั้งค่าส่วนตัว (Preferences) | ตั้งค่าความสนใจและการแจ้งเตือนต่างๆ | MVP |
| 1.5.6 | การตั้งค่าการแจ้งเตือน (Notification Settings) | เลือกรับหรือไม่รับการแจ้งเตือนแต่ละประเภท | MVP |

### 1.6 การยืนยันตัวตน (Authentication)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 1.6.1 | ล็อกอินผ่านไลน์ (LINE Login / Auto Login) | เข้าใช้งานอัตโนมัติผ่านแอป LINE ง่ายและรวดเร็ว | POC/MVP |
| 1.6.2 | ผูกเบอร์โทรศัพท์ (Mobile Binding) | ยืนยันตัวตนด้วยเบอร์มือถือ (OTP) เพื่อความปลอดภัยและกู้คืนบัญชี | MVP |

| 1.7.4 | รองรับหลายภาษา (Multi-language) | เมนูและเนื้อหารองรับทั้งภาษาไทยและอังกฤษ | MVP |

## 2. เว็บพันธมิตร (Partner Mobile Site)

Web Application ในรูปแบบ Mobile Site สำหรับร้านค้าพาร์ทเนอร์ และเจ้าหน้าที่จัดกิจกรรม

### 2.1 สแกน QR และตรวจสอบ (QR Scanner)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 2.1.1 | ระบบสแกน QR แบบผสม (Hybrid QR Scanner) | สแกน QR ตัวเดียว รองรับทั้งการให้แต้มและการตัดแต้มแลกของ | POC/MVP |
| 2.1.2 | การยืนยันตัวตนผู้ใช้ (User Verification) | ตรวจสอบข้อมูลลูกค้าก่อนทำรายการ | POC/MVP |
| 2.1.3 | การให้แต้ม (Manual Point Award) | ระบบกดให้แต้มลูกค้า เมื่อทำภารกิจสำเร็จ (เหตุการณ์: ใช้บริการครบยอด) | POC/MVP |
| 2.1.4 | การแลกของรางวัล (Coupon Redemption) | สแกนคูปองลูกค้าเพื่อบันทึกการใช้สิทธิ์รับของรางวัล | MVP |
| 2.1.5 | ประวัติการทำรายการ (Transaction History) | ดูรายการย้อนหลังที่ร้านค้าเคยทำไป | MVP |

### 2.2 การตรวจสอบภารกิจ (Mission Verification)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 2.2.1 | การตรวจสอบอัตโนมัติ (Auto Verification) | ระบบตรวจสอบเงื่อนไขและอนุมัติทันที (ถ้าตั้งค่าไว้) | MVP |
| 2.2.2 | การตรวจสอบโดยพาร์ทเนอร์ (Manual Verification) | เจ้าหน้าที่กดอนุมัติงานที่ต้องใช้คนดู (เช่น ดูรูปถ่าย) | MVP |
| 2.2.3 | การปฏิเสธงาน (Rejection with Reason) | กดยกเลิกภารกิจพร้อมระบุเหตุผลแจ้งลูกค้า | MVP |
| 2.2.4 | การอนุมัติแบบกลุ่ม (Batch Approval) | กดอนุมัติหลายรายการพร้อมกันในคราวเดียว | MVP |

### 2.3 แดชบอร์ดพันธมิตร (Partner Dashboard)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 2.3.1 | สรุปผลรายวัน (Daily Stats) | ดูยอดลูกค้าและการใช้งานรายวัน/รายสัปดาห์ | MVP |
| 2.3.2 | สรุปยอดแต้ม (Points Summary) | ดูจำนวนแต้มที่แจกไปและแต้มที่ถูกนำมาแลก | MVP |
| 2.3.3 | ภารกิจยอดฮิต (Top Missions) | ดูว่าลูกค้าชอบทำภารกิจไหนมากที่สุด | MVP |
| 2.3.4 | การส่งออกรายงาน (Export Report) | ดาวน์โหลดข้อมูลสรุปออกมาเป็นไฟล์ Excel/CSV | MVP |

## 3. ระบบจัดการหลังบ้าน (Admin Console)

ระบบเว็บไซต์สำหรับแอดมิน ผู้ดูแลระบบ และทีมการตลาด เพื่อจัดการภาพรวมทั้งหมด

### 3.1 แดชบอร์ดภาพรวม (Real-Time Dashboard)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 3.1.1 | ผู้ใช้งานขณะนี้ (Active Users) | กราฟแสดงจำนวนคนที่กำลังใช้งานระบบอยู่ตอนนี้ | MVP |
| 3.1.2 | การหมุนเวียนแต้ม (Point Circulation) | ดูยอดแต้มรวมในระบบ (แจกไปเท่าไหร่ ใช้ไปเท่าไหร่) | MVP |
| 3.1.3 | อัตราความสำเร็จ (Completion Rate) | เปอร์เซ็นต์คนทำภารกิจสำเร็จ | MVP |
| 3.1.4 | ผลงานพาร์ทเนอร์ (Partner Performance) | ดูยอดการใช้งานของแต่ละร้านค้า | MVP |
| 3.1.5 | ความคุ้มค่า (ROI Tracking) | คำนวณความคุ้มค่าของการแจกแต้ม | MVP |
| 3.1.6 | กราฟรูปแบบต่างๆ (Data Visualization) | แสดงผลข้อมูลในรูปแบบแผนภูมิที่ดูง่าย | MVP |

### 3.2 จัดการภารกิจ (Mission Management)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 3.2.1 | สร้างภารกิจ (Create Mission) | สร้างโจทย์ใหม่ๆ (เช่น ไปสถานที่, ถ่ายรูป, ซื้อของ) | POC/MVP |
| 3.2.2 | กำหนดแต้มรางวัล (Point Settings) | ตั้งค่าว่าทำเสร็จแล้วจะได้กี่แต้ม | POC/MVP |
| 3.2.3 | กำหนดผู้มีสิทธิ์ (Eligibility) | ตั้งเงื่อนไขว่าใครเล่นได้บ้าง (เช่น เฉพาะ VIP) | MVP |
| 3.2.4 | กำหนดระยะเวลา (Time Period) | ตั้งวันเริ่มและวันสิ้นสุดกิจกรรม | MVP |
| 3.2.5 | ติดตามผล (Performance Tracking) | ดูว่าคนสนใจเล่นภารกิจนี้มากน้อยแค่ไหน | MVP |

### 3.3 จัดการพื้นที่จีโอเฟนซ์ (Polygon & Geofence Manager)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 3.3.1 | นำเข้าพื้นที่ (Import KML) | อัปโหลดไฟล์แผนที่พื้นที่กิจกรรมจาก Google Earth/Maps | MVP |
| 3.3.2 | พื้นที่หลายจุด (Multi-Polygon) | วาดพื้นที่กิจกรรมได้หลายจุดพร้อมกัน | MVP |
| 3.3.3 | ทดสอบพิกัด (Test Tool) | เครื่องมือตรวจสอบว่าพิกัดนี้อยู่ในเขตที่ตั้งไว้หรือไม่ | MVP |
| 3.3.4 | แสดงบนแผนที่ (Map Visualization) | ดูขอบเขตพื้นที่กิจกรรมทั้งหมดบนหน้าจอแผนที่ | MVP |

### 3.4 จัดการของรางวัล (Reward Catalog Management)

| # | ฟีเจอร์ | คำอธิบาย | Phase |
|---|---------|----------|-------|
| 3.4.1 | จัดการรางวัล (Manage Rewards) | เพิ่ม ลบ แก้ไข รายการของรางวัล | MVP |
| 3.4.2 | กำหนดราคาแต้ม (Cost Settings) | ตั้งราคาว่าต้องใช้แต้มแลกเท่าไหร่ | MVP |
| 3.4.3 | กำหนดเวลา (Availability) | ตั้งช่วงเวลาที่เปิดให้แลกได้ | MVP |
| 3.4.4 | รูปภาพและรายละเอียด (Content) | ใส่รูปสวยๆ และคำบรรยายสินค้า | MVP |
| 3.4.5 | รองรับ 2 ภาษา (Bilingual Support) | ใส่ข้อมูลได้ทั้งไทยและอังกฤษ | MVP |
| 3.4.6 | สถิติการแลก (Redemption Stats) | ดูว่าของชิ้นไหนคนแลกเยอะสุด | MVP |
| 3.4.7 | ตัดสต็อกของ (Inventory Management) | นับจำนวนของที่เหลืออยู่ (สำหรับของที่จับต้องได้) | MVP |

### 3.5 จัดการพันธมิตร (Partner Management)

| # | ฟีเจอร์ | คำอธิบาย | Phase |

## 8. หมายเหตุ (Notes)

### คำแนะนำเพิ่มเติม (Recommendations)

1. **ล็อกอินด้วยไลน์เป็นหลัก (LINE-First Authentication)** - ใช้ LINE ID ง่ายต่อคนไทย
2. **ระบบ QR แบบผสม (Hybrid QR System)** - สแกนโค้ดเดียวทำได้ทุกอย่าง ลดความสับสน
3. **โอนแต้มแบบมีกติกา (Point Transfer with Controls)** - โอนได้แต่ต้องคุมเข้ม ป้องกันการขายแต้ม

### ฟีเจอร์ที่ไม่แนะนำ (Not Recommended Features)

1. **โอนแต้มให้เพื่อน (P2P Transfer)** - เสี่ยงต่อการโกงและขายแต้ม
2. **ระบบขายไอเทม (In-App Marketplace)** - ทำให้กลายเป็นเกมเติมเงิน (Pay-to-Win) ผิดวัตถุประสงค์ Engagement
